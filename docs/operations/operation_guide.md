# Guía de operación diaria

## 1. Comprobaciones de inicio de jornada

- revisar que la aplicación esté accesible
- revisar número de equipos offline
- revisar alarmas críticas abiertas
- comprobar que el histórico sigue creciendo
- verificar que no existe backlog anómalo de recuperación

## 2. Comprobaciones semanales

- revisar espacio libre en disco
- revisar tiempos de respuesta
- revisar crecimiento de base de datos
- revisar integridad de backups
- revisar tendencia de incidencias de comunicación

## 3. Comprobaciones tras cambios de configuración

- validar sintaxis de CSV
- validar accesibilidad IP y puertos
- confirmar aparición del activo en interfaz
- confirmar que el activo genera lecturas
- confirmar que las variables historizadas se insertan correctamente
